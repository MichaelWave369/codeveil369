# CodeVeil369 Architecture

CodeVeil369 models a layered path from structured light to symbolic rendering:

```text
L = laser / diffraction field
E = eye and entoptic transform
N = neural / perceptual toy model
G = glyph attractor and symbolic binding
C = perceived code-field
```

Core formula:

```text
C = G(N(E(L)))
```

## Runtime layers

1. **Structured Field** — simulated wavelength, diffraction, speckle, and interference-like patterns.
2. **Entoptic Transform** — vignette, macular-style radial effects, internal visual filtering, and persistence.
3. **Neural Field Toy Model** — geometry, tunnels, spirals, lattice effects, time elasticity, and visual folding.
4. **Glyph Attractor** — code-like symbols emerge from the visual field as symbolic overlays.
5. **Material Code Chamber** — wall, water, quartz, bamboo, magnet, and observer materials render different symbolic code families.
6. **Ledger Panel** — explicitly marks the app as simulation/art, not external proof.

## Design principle

The glyphs should feel grown from the field, not pasted on top. The app should stay mystical in feeling while remaining honest about claim boundaries.
